# Philosophy and Architecture

Bijou is not only a component catalog. The repo has an explicit design
stance about runtime truth, shell honesty, portability, and the role of
documentation as a proving surface.

## Core themes

- runtime truth beats vanished contracts
- host-specific APIs belong behind adapters
- docs should exercise the real framework
- shell chrome should teach rather than obscure

## What is still missing

The key doctrine and architecture documents already exist in `docs/`,

release blocker is to bring those documents into this section in a
