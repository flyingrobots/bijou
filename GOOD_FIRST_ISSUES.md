# Good First Issues

A small starting set for contributors:

1. Add a one-paragraph onboarding guide entry for another example in `examples/`.
2. Improve one package README that is missing setup or first-run output.
3. Add a focused `docs/specs/*.spec.json` acceptance criterion for an existing
   backlog-facing feature.
4. Add an environment-variable note to a GUIDE where missing and keep it sorted in
   `.env.example`.
5. Improve a README code sample to show the expected exit behavior for `q`.
