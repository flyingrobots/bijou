# DF-022 — Build Prose Docs Reader And Top-Level DOGFOOD Nav

Legend: [DF — DOGFOOD Field Guide](../../legends/DF-dogfood-field-guide.md)

## Idea

DOGFOOD needs a real docs-site shell, not just the current component
family explorer.

`4.1.0` should not ship until DOGFOOD can visibly navigate between
top-level docs sections and read prose-style docs pages inside the app.

## Why

Right now DOGFOOD is still mostly:

- landing screen
- component families
- component docs
- variants

That is a strong Components section, but it is not yet a terminal docs
site for Bijou.

## Ready when

- DOGFOOD has top-level section navigation beyond the component explorer
- a prose-doc reader exists inside the app
- the current components experience is clearly one section inside the
  broader docs shell instead of the whole product

## Status

`4.1.0` blocker spawned by
[DF-021](../../design/DF-021-shape-dogfood-as-terminal-docs-system.md).
