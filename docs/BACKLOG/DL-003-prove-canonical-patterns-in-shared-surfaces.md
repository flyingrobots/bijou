# DL-003 — Prove Canonical Patterns in Shared Surfaces

Legend: [DL — Design Language](/Users/james/git/bijou/docs/legends/DL-design-language.md)

## Idea

DL-002 makes the pattern and block language explicit.

The next step should prove that those patterns are not just prose by applying them to more shared surfaces:

- selected rows outside preference lists
- drawer and inspector rhythm
- notice rows and other shell-adjacent collections
- future block candidates that should inherit the same spacing, ownership, and overflow language

## Why

Pattern language only matters if it changes shared surfaces.

This cycle should convert the most credible documented patterns into more reusable implementation surfaces instead of leaving them trapped inside one proving context.
