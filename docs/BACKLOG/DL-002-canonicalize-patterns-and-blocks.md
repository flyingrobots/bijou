# DL-002 — Canonicalize Patterns and Blocks

Legend: [DL — Design Language](/Users/james/git/bijou/docs/legends/DL-design-language.md)

## Idea

DL-001 captures the doctrine layer.

The next design-language step should turn that doctrine into explicit reusable patterns and, later, blocks:

- canonical selection patterns
- list and preference patterns
- shell drawer and overlay patterns
- onboarding and explainability patterns
- eventual reusable blocks like app frame, login flows, and guided flows

## Why

Doctrine without patterns stays abstract.

This cycle should define the bridge from:

- north-star doctrine

to:

- reusable component/system patterns
- later block libraries

Backlog spawned by the retrospective for DL-001.
