# Legacy Specs

This directory is a legacy planning surface.

Existing files may remain here until they are migrated, superseded, or archived.

Individual `*.spec.json` files here are historical reference artifacts, not
the current source of queue truth.

## Rule

Do **not** start new work here.

New work should use:

- `docs/method/backlog/` for backlog capture
- `docs/design/` for active cycle design docs
- `tests/cycles/<cycle>/` for cycle-owned tests

If a legacy spec becomes relevant again, pull the active work into METHOD
first and then either migrate or explicitly retire the old spec from there.
