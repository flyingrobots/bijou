# Legends

Legends are the thematic umbrellas that group related cycles.

Each legend should describe:

- the overall goal
- the human users
- the agent users
- the hills
- the relevant invariants
- the latest completed cycle
- the live backlog that still matters

Every invariant should be protected by at least one legend, and the
invariant docs should name those protecting legends explicitly.

Cycles should link back to their legend doc, and legend docs should not
pretend shipped backlog items are still upcoming work.
